# Render Information in Its Shape

*A 10-minute talk. One idea per section.*

## The problem

We turn structured thinking into walls of prose. A status update becomes five
paragraphs. A decision becomes a buried sentence. The reader has to reconstruct
the structure we already knew.

## The thesis

Information has a shape. Metrics are a dashboard. A process is a sequence. A
comparison is a table. The medium should match the shape — not flatten
everything into the same gray text.

## Why prose loses

Prose hides the scan path. To find the one number that matters, the reader
reads everything. Density drops, time-to-answer rises, and the important signal
sits at the same visual weight as the filler.

## What "shape" means

- Numbers want to be compared → KPI tiles and charts.
- Steps want an order → numbered timelines.
- Options want to be weighed → side-by-side tables.
- Warnings want to interrupt → callouts.

## The shift

Stop rendering documents. Start rendering the *thing the document is about*.
The same source can become a living page where each piece of information is in
the format that makes it instantly legible.

## What you get

Faster scanning, fewer misreads, and content that respects the reader's time.
The author writes once in Markdown; the shape is applied on the way out.

## Call to action

Take one document you wrote this week. Ask: what shape is each part? Then render
it that way. Measure the time it takes a colleague to find the key fact —
before and after.
