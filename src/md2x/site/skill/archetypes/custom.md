## Archetype: custom

For when the user supplies their own direction — `style_prompt` is the brief and overrides the defaults below.

**Shell:** sidebar. **Render mode:** blocks (default — change it if the prompt implies otherwise).

**Structure.** Read `style_prompt` first and treat it as law: it dictates layout, tone, shell, sections, and order. Map its intent onto Thariq's principle — render the content in the shape the prompt asks for, not a Markdown dump. Choose whichever blocks fit the requested structure; if the prompt calls for interactivity, motion, or a tool, switch to hybrid/full and pick the matching artifact. When the prompt is silent on something, fall back to the closest standard archetype's instincts rather than improvising wildly.

**Favor.** Whatever artifact best realizes the prompt — any pattern in the vocabulary is fair game. Let the prompt pick the render mode.

**Avoid.** Don't override or dilute explicit `style_prompt` instructions with these defaults, and don't ignore parts of it. Still stay faithful to the source content's facts — style is free, truth is not.
